## 2. SQL Injection
=> 웹 애플리케이션의 뒷단에 있는 Database에 질의하는 과정 사이에
   일반적인 값 외에 악의적인 의도를 갖는 구문을 습입하여 공격자가
   원하는 SQL 쿼리문을 실행하는 기법

- 방어 방법
1. 입력받는 특수문자를 검사하여 방어한다.
2. 오류 발생시 에러 메시지를 표시하면 안된다.
3. Statement대신  Preparestatement를 사용한다.
4. Hash function을 사용한다.
